## React 에서의 TypeScript 기능 이용
### State / Props
- TS를 적용 했을때는 컴포넌트 프롭의 타입을 정의해야 한다. Component의 prop이 JS 객체라는것을 배웠는데, 그러므로 TS를 적용했을때 프롭의 타입을 정의하기 위해 `type` 또는 `interface`를 이용할수 있다.

HelloComponent.tsx를 생성하고 임시 string data만 return. App.tsx에서 HelloComponent를 집어넣기.

```ts
export type HelloProps = {
  name: string;
  age: number;
}
```
이상과 같이 types.ts를 통해 자료형들을 전부 명시(추후 프로젝트 때도 사용할 예정).
- type export의 경우에 앞에 적어주겠습니다(Component와 동일하게).

```tsx
// App.tsx
import HelloComponent from './HelloComponent'
import './App.css'

function App() {

  return (
    <>
      <HelloComponent name='김영' age='twenty' />
    </>
  )
}

export default App

// HelloComponent.tsx
import {HelloProps} from './types.ts';

export default function HelloComponent({ name, age } : HelloProps ) {

  return(
    <h1>
      안녕하세요, {name} 님. {name} 님은 올해 {age} 살입니다.
    </h1>
  );
}
```

- 이상의 코드라인과 이미지를 참조하였을 때 중요한 점은 이하와 같다.
1. App.tsx에 빨간줄이 떠있기는한데 브라우저에 값이 출력된다. 이상의 이유는 TypeScript가 JSX의 보조이기 때문에 경고만 띄운 상태에 해당하기 때문이다. 즉 브라우저를 켜기 전에 잠재적인 오류를 잡아낼 수 있도록 하는 것이 TypeScript의 첫번째 존재 이유에 해당한다.

```ts
// 매개변수가 없는 함수 (call1() 유형)
export type = {
  name: string;
  age?: number
  fn: () => void;
}

// 매개변수가 있는 함수(call3() 유형)
export type = {
  name: string;
  age?: number
  fn: (msg: string) => void;
}
```

- types.ts 형태로 미리 분리한 예시를 보였지만, 추후 수업에서는 Component내에 집어넣었다가 분리하는 과정을 연습하여 일종의 객체지향형태로 작성해보는 과정을 frontend 버전으로 수업한다.

- 이상의 경우에서 props 타입을 정의하는 방법에 대해 배웠다. 이번에는 state 관련 수업. 'useState()`를 생각해보면 내부의 initialValue가 뭐냐에 따라서 추후 상황이 결정 된다.
```tsx
// 문자열 초기값
const [ name, setName ] = useState('');
// 숫자 초기값
const [ age, setAge ] = useState(0);
// boolean 초기값
const [ isReady, setReady ] = useState(false);
```

그 상황에서 `setAge('ten')`이라고 입력하면 오류가 발생하는데, TS를 배운 입장에선 추론할수있는게 useState react hook을 사용하게 됐을때, initialCalue에 대한 타입 추론이 일어난다고 생각할수 있다.

```tsx
  const [ age,setAge ] = useState(0);
  // initialValue는 0이지만, setAge로 값을 변경할 수 있다.
  // setAge('ten');
```
와 같이 작성할 경우 타입 추론을 통한 매개변수가 number임에도 불구하고 함수의 매개변수가 string이기 때문에 타입 불일치로 인한 브라우저 return이 실패한다. 근데 react hook은 jsx에서도 있었기 때문에 아까 위의 예시와 달리 아예 결과값이 나오지않는다.

- 상태의 타입을 '명시적으로' 정의하는 것도 가능하다. 특히 상태를 null 또는 undefined로 초기화 하려면 특히 요구 된다.

```tsx
const [message, setMessage] = useState<string | undifined>(undefined);
```

- 이상의 코드는 함수에서의 매개변수 타입의 명시적 정의와 방식이 다르다. 그 이유는 useState라는 react hook이 tsx에서만 적용되는것이 아니기도 하고, 이미 정의된 함수를 호출하여 message / setMessage 배열을 return한 것이기 때문에 `함수명(매개변수1:'string', 매개변수2:number) : void;` 와 같은 이전에 배웠던 방법이 아니라 `<>` 재네릭을 썻다는 점이다. 이는 ts자체가아니라 ts와 react가 결합된 tsx 상에서의 문법이다.

- 하지만 이건 상태가 하나짜리일때에 가깝고, 다수의 타입을 명시해야 한다면 여태까지 해온 것 처럼 type / interface를 사용할수도 있다.

```tsx
type User = {

  age: number;

export default Example() {
  const [user, setUser] = useState<user>({})

  // null이 가능하도록 하려면
  const [user, setUser] = useState<User | null>(null);

  return 중략
}
```

## Promise
- 비동기 연산을 처리하는 고전적인 방법은 연산의 성공 또는 실패에 콜백 함수를 달아주는것이다. 연산이 성공 했을경우에는 success 함수가 호출되고, 실패하면 failure 함수가 호출되는 방식이 된다.

```tsx
function doAsync(sucess, failure) {
  if(SUCCEED) {
    success(resp);
  } else {
    failure(err);
  }
}

function success(response) {
  // 응답을 가지고 수행하게 되는 로직
}

function failure(error){
  // 오류 처리 로직을 작성
}

// 함수 호출
doAsyncCall(success, failure);
```

- 이상의 예시는 Promise가 도입되기 전 방식이다. Promise는 JS에서 비동기 프로그래밍의 기본요소에 해당한다.
- Promise의 정의 : 비동기 연산의 결과를 나타내는 JS 객체.
- 이를 이용하게 되면 위에서 처럼 성공 실패에 대한 함수들을 일일이 정의할 필요가 없어지기 때문에 비동기 호출을 실행할때, 코드가 단순화 되는 장점이 있다. 그리고 콜백이 중첩되지않기 때문에 callback hell을 겪지않아도 된다.

참조 : https://developer-haru.tistory.com/57
- 이하의 예시에서 비동기 호출을 수행하고, 응답이 반환되면 then() 메서드 내부의 콜백 함수가 실행되어 응답을 argument로 받게 된다.
```js
doAsyncCall().then(response => //응답 성공이니까 그걸로 로직수행) //success() 함수 부분을 정의하고 호출하는 단계가 한줄로 축약된다.
```

then() 메서드는 Promise 객체를 return한다. 그리고 이 Promise는 세가지 상태중 하나에 해당하게 된다.
1. Pending(대기) : 초기상태
2. Resolved / FulFilled(이행) : 작업성공
3. Rejected(거부) : 작업 실패

이하에서 예시 삽입
```js
const myPromise = new Promise((resolve, reject) => {
  setTimeout(() => {
    resolve('Hello');   // 'Hello'라는 애를 가지고 뭔가를 한다. 정의하지않았음.
  }, 500);
});
```

- Promise 객체가 생성될때와 타이머가 실행되는 동안에 Promise는 대기(Pending) 상태에 있다. 500밀리초가 지나면 'Hello'라는 값으로 resolve 함수가 호출되고, Promise는 이행(resolved) 상태가 된다. 만약에 오류가 발생하게 되면 .then()을 쓰지않았지만 Promise 상태가 거부(rejected)rk ehlsek. -> catch()를 통해 해결한다.

- .then() 메서드를 통해 인스턴스를 서로 연결하여 다수의 비동기 작업을 순차적으로 진행시킬수 있다. then: 그리고 나서, 의 의미로 받아들어주면 된다.
```js
doAsyncCall()
  .then(response => //응답에서 일부 데이터만 가지고 오기(예를 들어 jsonplaceholder에서 제목만 가지고 오거나, 제목+내용만 가지고 오거나, 제목+내용+작성자를 가져오는 등)
  )
  .then(data => // 그 가지고온 데이터를 가지고 브라우저에 뿌리는 등의 로직
  )
```

- 오류발생시엔 Promise에 오류 처리를 추가할수 있다.
```js
doAsyncCall()
  .then(response => //로직수행
  )
  .then(data => // 그 가지고온 데이터를 가지고 브라우저에 뿌리는 등의 로직
  )
  .catch(error => console.lon(error)) // 근데 then() 수행중에 뭔가 오류나면 콘솔에 오류내용 확인 -> response에서 오류났으면 맨 처음 부분 수정해야하고, data부분에서 오류나면 그 다음부분을 수정하면 될것이다.
```

### async / await

- 비동기 호출을 처리하는 .then() 보다 더 최신 방법에 해당한다. 애초에 Promise를 기반으로 하는데, 기본적으로 async / await을 이용하려면 await 표현식을 포함할 수 있는 async() 함수를 정의해줘야 한다.

```js
const doAsyncCall = async() => {
  const response = await fetch(`http://sampleapi.com`);
  const data = await response.json();
  // 그 다음 이상에서 정의된 data 변술르 가지고 뭐든 하면 된다.
}
```

- fetch() 함수 자체도 return 타입이 Promise 객체이다. 그런데 위에 처럼 Promise가 return 되었음에도 .then() 메서드를 사용하는게 아니라 `await`키워드를 사용했다는 점이 차이가 있다.

- promise가 처음으로 도입되었을때에는 시간적인 차이를 나타내기 위해 then() 이라는 접속사를 활용한 메서드를 정의했었지만, 이후 발전과정에서 기존의 JS 문법으로 표현을 할 필요성을 느낀 개발자 들이 `async / await` 키워드를 추가 개발했다고 보면 된다.

- .then()을 도힙했다면 : chaining methods를 통한 순차 처리를 예시로 코드를 작성
- async / await을 도입했다면 : 비동기 처리를 하는 함수를 동기적으로 작성하는 코드를 작성

## fetch API 활용
- fetch API는 라이브러리 설치가 필요없다.
- fetch API는 호출하려는 리소스 경로를 필수 argument로 갖는 fetch() 메서드를 가지고 있다. 웹 요청의 경우는 argument로 서비스의 URL이 된다. 그래서 응답을 반환하는 간단한 GET 요청은 이하와 같다

```tsx
fetch(`https://someapi.com/${id}`)
  .then(response => respone.json())
  .then(data => console.log(data))
  .catch(error => console.log(error));
```
- 이상의 예시에서 첫번째 .then() 문에 전달되는 response는 요청이 성공했는지 확인하는데 이용할수 있는 ok 및 status 속성을 포함하는 Promise 객체이다. 응답상태가 2xx 형식이면 ok 가 되고, 속성 값은 true가 된다.,

```tsx
fetch(`https://someapi.com/${id}`)
  .then(response => {
    if(response.ok) {
      // 요청성공 -> 상태 2xx에 해당함.
    } else {
      // 문제 발생 예시 -> 오류응답
    }
  })
  .then(data => console.log(data))
  .catch(error => console.log(error));
```

POST 요청과 같은 HTTP 메서드를 이용하려면 fetch() 메서드의 두번째 argument가 필요하다. 두번째 argument는 여러요청을 정의 할수 있는 `객체`이다. 

- POST 요청 예시

```tsx
fetch(`https://someapi.com`,{method: "POST"})
  .then(response => respone.json())
  .then(data => console.log(data))
  .catch(error => console.log(error));
```

- 그런데 두번째 argument에 헤더를 추가하는 것도 가능하다. 이하에선 postman에서 간단하게 만들었던 것을 직접 명시하는 코드이다.
```tsx
fetch(`https://someapi.com`,{
  method: "POST",
  headers: {"Content-Type" : "application/json", },
  .then(response => respone.json())
  .then(data => console.log(data))
  .catch(error => console.log(error));
  })
```

- 마지막으로 우리가 todolist에서 localStorage 등에 저장하거나 가지고 올 때 stringify나 json을 썻었다.

```tsx
fetch(`https://someapi`,
  {
    method: "POST",
    headers: { "Content-Type" : "application/json"},
    body : JSON.stringify(data)
  }
)
  .then(response => respone.json())
  .then(data => console.log(data))
  .catch(error => console.log(error));
```

## axios 라이브러리 이용
- 지금 then() 메서드가 Promise에서도 쓰였고 fetchAPI 에서도 예시로 다 작성되었다. 그래서 처음 수업 받는 사람들이 혼란스러워 하는 경우가 많은데 .then() 대신에 async/await을 도입할수 있다. 그리고 fetch API는 HTTP 메서드들을 처리하기 위한 API로, 얘를 대체하는 라이브러리가 axios에 해당한다.

- 그러면 섞여서 나올수 있는게 .then() 대신에 async / await은 쓰지만 fetch API를 그대로 쓰고 있을수도 있고, axios를 쓰지만 then() 메서드를 그대로 쓸수도 있다.

- fetch API의 불편함을 일부 대체하는 외부 라이브러리인 axios를 도입하기 위해서는 터미널에 `npm install axios`가 요구 된다.
- axios 라이브러리 내에는 JSON 데이터의 자동변환을 지원하기 때문에 json() 함수가 필요하지않다.
```jsx
import axios from 'axios';

axios.get(`http://someapi.com`)
.then(response => console.log(response))  // fetch API 에서는 response => response.json() 이였음.
.catch(error => console.log(error))
```

- POST 요청 예시
```jsx
axios.post(`http://someapi.com`, {newObject})
.then(response => console.log(response))
.catch(error => console.log(error))
```

- 이상이 axios + then() 메서드 예시이다.

```jsx
const response = await axios({
  method : "POST",
  url : `https://myapi.com/api/cars`,
  headers : {"Content-Type" : "application/json"},
  data : { brand : '현대', model : '소나타'},

```

`main.temp` 거나 `main["temp"]`로 작성하면 된다. 하지만 description과 icon의 경우에는 key-value로 속기 쉬울수 있지만 weather의 value가 배열이고, 거기의 0번 element가 JS 객체라는 것을 확인할수 있다. 결과적으로 icon의 값을 frontend의 상태에 저장하기 위해서는 `weather[0].icon` 이거나 `weather[0]['icon']`이 되어야 한다.

- 첫 렌더링 시에 fetch()를 시도해서 그 값으로 고정시킬 예정. 상태가 바뀔때마다 확인하는 것이 아니기 때문에 `useEffect(()=>{},[])`에서 `,[]` 부분이 그 부분을 통제한다.

- 그러면 첫 렌더링시에 useEffect() 내부의 콜백 함수에서 setWeather() 함수를 호출해서 값을 대입해야된다. 그리고 useEffect()의 콜백 함수에서 fetch()를 수행하고, Promise 객체가 나오게 되면 걔를 json으로 바꿔주고, 그 json값에서 필요한 key-value 조합을 꺼낸 다음에 상태에 저장해야한다.

```jsx
import { useState,useEffect } from 'react'
import './App.css'

function App() {
const [weather, setWeather] = useState({
  temp : '',
  desc : '',
  icon : '',
});

  useEffect(() => {
  fetch(`https://api.openweathermap.org/data/2.5/weather?q=Busan&units=Metric&APIkey=6dc2ff4a2ab5f77db42e6e8744f72343`)
  .then(res => res.json())
  .then(data => {
    setWeather({
      temp: data.main.temp,
      desc: data.weather[0].description,
      icon: data.weather[0].icon
    })
  })
  .catch(err => console.log(err));
  }, []);

  return (
    <>
      
    </>
  )
}

export default App
```

- 특히 useEffect() 부분 전체에 해당하는 부분을 해석해 볼만한 가치가 있다. 어디까지가 useEffect()의 영역인지, 어디가 fetch()의 영역인지, 그 다음 then은 어디까지인지 등을 확인 할수 있다.

6. icon을 확인하면 json에서도 그렇고 기본적으로 string 자료형이다. 그림이 아니라서 날씨 아이콘을 표시하려면 아이콘 코드 앞에 `https://openweather.org/img/wn/`을 표시하고, 맨 마지막에 endpoint로 json의 icon값을 가지고 와야 한다.

```jsx
import { useState,useEffect } from 'react'
import './App.css'

function App() {
const [weather, setWeather] = useState({
  temp : '',
  desc : '',
  icon : '',
});

  useEffect(() => {
  fetch(`https://api.openweathermap.org/data/2.5/weather?q=Busan&units=Metric&APIkey=6dc2ff4a2ab5f77db42e6e8744f72343`)
  .then(res => res.json())
  .then(data => {
    setWeather({
      temp: data.main.temp,
      desc: data.weather[0].description,
      icon: data.weather[0].icon
    })
  })
  .catch(err => console.log(err));
  }, []);

  if (weather.icon) {
      return (
      <>
        <p>기온 : {weather.temp}</p>
        <p>설명 : {weather['desc']}</p>
        <p>아이콘 string : {weather.icon}</p>
        <img src={`https://openweathermap.org/img/wn/${weather.icon}@2x.png`} alt='날씨 아이콘'/>
      </>
  )
  } else {
    return <h1>loading...⏰</h1>
  }


}
```
