## 1. React에서 TypeScript 활용

### 1-1. Props 타입 정의

컴포넌트의 Props는 JS 객체이므로, `type` 또는 `interface`로 타입을 명시한다.

```ts
// types.ts
export type HelloProps = {
  name: string;
  age: number;
}
```

```tsx
// HelloComponent.tsx
import { HelloProps } from './types.ts';

export default function HelloComponent({ name, age }: HelloProps) {
  return (
    <h1>안녕하세요, {name} 님. {name} 님은 올해 {age} 살입니다.</h1>
  );
}
```

> **핵심 포인트**: 타입 불일치 시 TypeScript는 브라우저 실행 전에 경고를 띄워준다.  
> 이것이 TypeScript의 첫 번째 존재 이유 — **잠재적 오류를 사전에 감지**.

---

### Props에 함수 타입 포함하기

```ts
// 매개변수 없는 함수
export type MyProps = {
  name: string;
  age?: number;       // ? → 선택적(optional) 속성
  fn: () => void;
}

// 매개변수 있는 함수
export type MyProps = {
  name: string;
  age?: number;
  fn: (msg: string) => void;
}
```

---

### 1-2. State 타입 정의

`useState()`는 **초기값으로 타입을 자동 추론**한다.

```tsx
const [name, setName] = useState('');       // string으로 추론
const [age, setAge] = useState(0);          // number로 추론
const [isReady, setReady] = useState(false); // boolean으로 추론
```

> `setAge('ten')` 처럼 타입이 다른 값을 넣으면 오류 발생.  
> Props와 달리 경고가 아니라 **아예 결과값이 나오지 않는다** (React Hook이기 때문).

---

### 명시적 타입 정의 (제네릭 사용)

초기값을 `null` 또는 `undefined`로 설정할 때는 제네릭(`<>`)으로 명시한다.

```tsx
const [message, setMessage] = useState<string | undefined>(undefined);
```

> `함수명(매개변수: 타입)` 방식이 아니라 `<>` 제네릭을 쓰는 이유:  
> `useState`는 이미 정의된 함수를 **호출**하는 것이기 때문. (tsx 고유 문법)

---

### 객체 상태 타입 정의

```tsx
type User = {
  id: number;
  name: string;
  age: number;
}

const [user, setUser] = useState<User | null>(null);
```

## 2. React RESTful API

### 2-1. Promise

비동기 연산의 결과를 나타내는 JS 객체.  
콜백 함수를 직접 정의·전달하던 방식을 단순화하고, **Callback Hell을 방지**한다.

#### Promise의 3가지 상태

| 상태 | 설명 |
|------|------|
| Pending (대기) | 초기 상태 |
| Fulfilled (이행) | 작업 성공 |
| Rejected (거부) | 작업 실패 |

#### `.then()` 체이닝 예시

```js
doAsyncCall()
  .then(response => /* 데이터 가공 */)
  .then(data => /* 브라우저에 렌더링 */)
  .catch(error => console.log(error));
```

---

### 2-2. async / await

Promise 기반이지만, **비동기 처리를 동기적으로 읽히게** 작성하는 방법.

```js
const doAsyncCall = async () => {
  const response = await fetch('http://sampleapi.com');
  const data = await response.json();
  // data를 가지고 로직 수행
}
```

| 방식 | 특징 |
|------|------|
| `.then()` | 메서드 체이닝으로 순차 처리 |
| `async/await` | 동기 코드처럼 읽히는 비동기 처리 |

> 둘은 대체 관계 — `fetch API + async/await` 또는 `axios + .then()` 등 혼합 사용 가능.

---

### 2-3. fetch API

별도 설치 없이 사용 가능한 내장 API.

#### GET 요청

```tsx
fetch(`https://someapi.com/${id}`)
  .then(response => response.json())
  .then(data => console.log(data))
  .catch(error => console.log(error));
```

#### POST 요청 (헤더 + body 포함)

```tsx
fetch('https://someapi.com', {
  method: 'POST',
  headers: { 'Content-Type': 'application/json' },
  body: JSON.stringify(data)
})
  .then(response => response.json())
  .then(data => console.log(data))
  .catch(error => console.log(error));
```

> `response.ok` → 요청 성공 여부 확인 (2xx 상태코드면 `true`)

---

### 2-4. axios 라이브러리

fetch API를 대체하는 외부 라이브러리. `npm install axios`로 설치.

#### fetch API vs axios 차이점

| 항목 | fetch API | axios |
|------|-----------|-------|
| JSON 변환 | `.json()` 직접 호출 필요 | **자동 변환** |
| JS 객체 직렬화 | `JSON.stringify()` 필요 | **자동 처리** |
| 설치 | 불필요 (내장) | `npm install axios` |

#### GET 요청

```jsx
import axios from 'axios';

axios.get('http://someapi.com')
  .then(response => console.log(response))
  .catch(error => console.log(error));
```

#### POST 요청

```jsx
axios.post('http://someapi.com', { newObject })
  .then(response => console.log(response))
  .catch(error => console.log(error));
```

## 3. 실습 — 날씨 앱 (Weather App)

OpenWeatherMap API를 활용하여 부산 날씨(기온 / 설명 / 아이콘)를 표시하는 앱 구현.

### 핵심 흐름

```
첫 렌더링
  → useEffect(()=>{}, []) 실행
    → fetch()로 API 호출
      → .then()으로 JSON 변환
        → 필요한 값 추출 후 setWeather()로 상태 저장
          → 리렌더링 → 화면 출력
```

### JSON 데이터 구조 분석

```json
{
  "main": { "temp": 10.99 },
  "weather": [
    { "description": "clear sky", "icon": "01d" }
  ]
}
```

> `weather`는 **배열**이므로 → `data.weather[0].icon` 으로 접근해야 한다.

### 최종 코드

```jsx
import { useState, useEffect } from 'react';
import './App.css';

function App() {
  const [weather, setWeather] = useState({
    temp: '',
    desc: '',
    icon: '',
  });

  useEffect(() => {
    fetch(`https://api.openweathermap.org/data/2.5/weather?q=Busan&units=Metric&APIkey=YOUR_KEY`)
      .then(res => res.json())
      .then(data => {
        setWeather({
          temp: data.main.temp,
          desc: data.weather[0].description,
          icon: data.weather[0].icon,
        });
      })
      .catch(err => console.log(err));
  }, []);

  if (weather.icon) {
    return (
      <>
        <p>기온 : {weather.temp}</p>
        <p>설명 : {weather.desc}</p>
        <img
          src={`https://openweathermap.org/img/wn/${weather.icon}@2x.png`}
          alt="날씨 아이콘"
        />
      </>
    );
  } else {
    return <h1>loading...⏰</h1>;
  }
}

export default App;
```

> 아이콘 이미지 URL 형식: `https://openweathermap.org/img/wn/{icon코드}@2x.png`
