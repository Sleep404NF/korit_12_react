## 1. PUT 요청 (Edit 기능)

### 구현 흐름
```
Edit 버튼 클릭
  → handleClickOpen() 실행
    → 기존 차량 데이터를 car 상태에 저장
      → Dialog 열림 (기존 데이터가 input에 표시)
        → 사용자가 수정 후 Save 클릭
          → mutate(carEntry) 실행
            → PUT 요청 → DB 업데이트
              → invalidateQueries → 목록 자동 갱신
```

## 4. 최종 코드

### types.ts

```ts
export type CarResponse = {
  brand: string;
  model: string;
  color: string;
  registrationNumber: string;
  modelYear: number;
  price: number;
  _links: {
    self: { href: string };
    car: { href: string };
    owner: { href: string };
  };
}

export type Car = {
  brand: string;
  model: string;
  color: string;
  registrationNumber: string;
  modelYear: number;
  price: number;
}

// PUT 요청용 — url과 car를 하나의 객체로 묶음
// React Query mutationFn은 매개변수를 하나만 받기 때문
export type CarEntry = {
  car: Car;
  url: string;
}
```

> `Car`와 `CarResponse`를 별개 타입으로 분리한 이유:  
> POST / PUT 요청 시 `_links`는 보내지 않아도 되므로 분리하는 것이 깔끔하다.

### carapi.ts

```ts
import { CarResponse, Car, CarEntry } from '../types'
import axios from 'axios'

// GET — ?size=100으로 Spring Boot 기본 페이지 제한(20개) 우회
export const getCars = async (): Promise<CarResponse[]> => {
  const response = await axios.get(
    `${import.meta.env.VITE_API_URL}/api/vehicles?size=100`
  );
  return response.data._embedded.cars;
}

// DELETE
export const deleteCar = async (link: string) => {
  const response = await axios.delete(link);
  return response.data;
}

// POST
export const addCar = async (car: Car) => {
  const response = await axios.post(
    `${import.meta.env.VITE_API_URL}/api/vehicles`, car
  );
  return response.data;
}

// PUT — Content-Type 헤더 명시
export const updateCar = async (carEntry: CarEntry): Promise<CarResponse> => {
  const response = await axios.put(carEntry.url, carEntry.car, {
    headers: { 'Content-Type': 'application/json' },
  });
  return response.data;
}
```

### AddCar.tsx

```tsx
import { Dialog, DialogTitle, DialogActions, Button } from "@mui/material"
import { Car } from "../types"
import { useState } from "react"
import { addCar } from "../api/carapi"
import { useMutation, useQueryClient } from "@tanstack/react-query"
import CarDialogContent from "./CarDialogContent"

export default function AddCar() {
  const queryClient = useQueryClient();

  const { mutate } = useMutation(addCar, {
    onSuccess: () => {
      queryClient.invalidateQueries(['cars']);
    },
    onError: (err) => console.log(err),
  });

  const [open, setOpen] = useState(false);
  const [car, setCar] = useState<Car>({
    brand: '', model: '', color: '',
    registrationNumber: '', modelYear: 0, price: 0
  });

  const handleChange = (event: React.ChangeEvent<HTMLInputElement>) => {
    setCar({ ...car, [event.target.name]: event.target.value });
  }

  const handleSave = () => {
    mutate(car);
    setCar({ brand: '', model: '', color: '', registrationNumber: '', modelYear: 0, price: 0 });
    setOpen(false);
  }

  return (
    <>
      <Button variant="contained" onClick={() => setOpen(true)}>새 차량 추가</Button>
      <Dialog open={open} onClose={() => setOpen(false)}>
        <DialogTitle>새로운 차량 등록</DialogTitle>
        <CarDialogContent car={car} handleChange={handleChange} />
        <DialogActions>
          <Button onClick={() => setOpen(false)} color="error">취소</Button>
          <Button onClick={handleSave} variant="contained" color="primary">저장</Button>
        </DialogActions>
      </Dialog>
    </>
  )
}
```

### EditCar.tsx

```tsx
import { useState } from "react";
import { CarResponse, Car, CarEntry } from "../types";
import { Dialog, DialogActions, DialogTitle, Button } from "@mui/material";
import CarDialogContent from "./CarDialogContent";
import { updateCar } from "../api/carapi";
import { useMutation, useQueryClient } from "@tanstack/react-query";

type FormProps = {
  cardata: CarResponse;
}

export default function EditCar({ cardata }: FormProps) {
  const [open, setOpen] = useState(false);
  const [car, setCar] = useState<Car>({
    brand: '', model: '', color: '',
    registrationNumber: '', modelYear: 0, price: 0
  });

  const queryClient = useQueryClient();
  const { mutate } = useMutation(updateCar, {
    onSuccess: () => {
      queryClient.invalidateQueries(['cars']);
    },
    onError: (err) => console.log(err),
  });

  // 모달 열기 전에 기존 데이터를 상태에 저장 → input에 기존값 표시
  const handleClickOpen = () => {
    setCar({
      brand: cardata.brand,
      model: cardata.model,
      color: cardata.color,
      registrationNumber: cardata.registrationNumber,
      modelYear: cardata.modelYear,
      price: cardata.price
    });
    setOpen(true);
  }

  const handleSave = () => {
    const url = cardata._links.self.href;
    const carEntry: CarEntry = { car, url };
    mutate(carEntry);
    setOpen(false);
  }

  const handleChange = (event: React.ChangeEvent<HTMLInputElement>) => {
    setCar({ ...car, [event.target.name]: event.target.value });
  }

  return (
    <>
      <Button size="small" variant="contained" onClick={handleClickOpen}>수정</Button>
      <Dialog open={open} onClose={() => setOpen(false)}>
        <DialogTitle>차량 변경</DialogTitle>
        <CarDialogContent car={car} handleChange={handleChange} />
        <DialogActions>
          <Button onClick={() => setOpen(false)} color="error">취소</Button>
          <Button onClick={handleSave} variant="contained" color="primary">저장</Button>
        </DialogActions>
      </Dialog>
    </>
  );
}
```

### Carlist.tsx

```tsx
import { useQuery, useMutation, useQueryClient } from "@tanstack/react-query";
import { getCars, deleteCar } from "../api/carapi";
import { DataGrid, GridColDef, GridCellParams, GridToolbar } from "@mui/x-data-grid";
import { Snackbar, IconButton, Tooltip } from "@mui/material";
import DeleteForeverRoundedIcon from '@mui/icons-material/DeleteForeverRounded';
import { useState } from "react";
import AddCar from "./AddCar";
import EditCar from "./EditCar";

export default function Carlist() {
  const [open, setOpen] = useState(false);
  const queryClient = useQueryClient();

  const { data, error, isSuccess } = useQuery({
    queryKey: ['cars'],
    queryFn: getCars
  });

  const { mutate } = useMutation(deleteCar, {
    onSuccess: () => {
      queryClient.invalidateQueries(['cars']);
      setOpen(true);
    },
    onError: (err) => console.error(err),
  });

  const columns: GridColDef[] = [
    { field: 'brand', headerName: 'Brand', width: 200 },
    { field: 'model', headerName: 'Model', width: 200 },
    { field: 'color', headerName: 'Color', width: 200 },
    { field: 'registrationNumber', headerName: 'RegistrationNumber', width: 200 },
    { field: 'modelYear', headerName: 'ModelYear', width: 200 },
    { field: 'price', headerName: 'Price', width: 200 },
    {
      field: 'edit',
      headerName: '',
      sortable: false,
      filterable: false,
      disableColumnMenu: true,
      renderCell: (params: GridCellParams) => <EditCar cardata={params.row} />
    },
    {
      field: 'delete',
      headerName: '',
      sortable: false,
      filterable: false,
      disableColumnMenu: true,
      renderCell: (params: GridCellParams) => (
        <Tooltip title="delete Car">
          <IconButton aria-label="delete" size="small"
            onClick={() => {
              if (confirm(`${params.row.brand}의 ${params.row.color} ${params.row.model}을(를) 삭제하시겠습니까?`)) {
                mutate(params.row._links.self.href);
              }
            }}
          >
            <DeleteForeverRoundedIcon />
          </IconButton>
        </Tooltip>
      )
    }
  ]

  if (!isSuccess) return <span>Loading...</span>
  if (error) return <span>자동차 데이터를 가져오던 중 오류가 발생했습니다.</span>

  return (
    <>
      <AddCar />
      <DataGrid
        rows={data}
        columns={columns}
        disableRowSelectionOnClick={true}
        getRowId={row => row._links.self.href}
        slots={{ toolbar: GridToolbar }}
      />
      <Snackbar
        open={open}
        autoHideDuration={2000}
        onClose={() => setOpen(false)}
        message='차량 정보가 삭제 되었습니다. 🗑️'
      />
    </>
  )
}
```

## 7. Frontend Application Protection (개념 정리)

> 수업에서 제목만 언급된 항목. 이후 OAuth2 / AWS 배포 수업과 연결될 내용.

프론트엔드는 코드가 브라우저에 그대로 노출되는 구조이기 때문에 백엔드와는 다른 방식의 보안 개념이 필요하다.

### 주요 개념

**XSS (Cross-Site Scripting)**
- 공격자가 악성 스크립트를 페이지에 삽입하는 공격.
- React는 `{}` 데이터 바인딩 시 자동으로 이스케이프 처리하여 기본적으로 방어한다.
- `dangerouslySetInnerHTML`은 이 방어를 우회하므로 사용 시 주의가 필요하다.

**CSRF (Cross-Site Request Forgery)**
- 사용자가 의도하지 않은 요청을 보내도록 유도하는 공격.
- Spring Security에서 CSRF 토큰을 발급하고 요청 시 검증하는 방식으로 방어한다.

**환경 변수 보안**
- `.env`의 `VITE_` 변수는 빌드 시 JS 번들에 포함되어 브라우저에서 확인 가능하다.
- API Key 같은 민감한 정보는 프론트에 두면 안되고 **백엔드를 통해 처리**해야 한다.
- 현재 프로젝트처럼 `localhost` 주소 정도는 괜찮지만, 실제 서비스 키는 절대 프론트에 노출하면 안된다.

**인증 / 인가 (Authentication / Authorization)**
- 다음 수업 예정인 OAuth2(구글 로그인)와 직접 연결되는 개념.
- 인증(Authentication): 사용자가 누구인지 확인 (로그인)
- 인가(Authorization): 해당 사용자가 무엇을 할 수 있는지 확인 (권한)
- 프론트에서는 로그인 상태에 따라 페이지 접근을 제한하는 **Protected Route** 패턴으로 구현한다.

```tsx
// Protected Route 개념 예시
function PrivateRoute({ children }) {
  const isLoggedIn = // 토큰 또는 세션 확인
  return isLoggedIn ? children : <Navigate to="/login" />
}

// 사용
<Route path="/admin" element={
  <PrivateRoute>
    <AdminPage />
  </PrivateRoute>
} />
```

## 8. 자주 발생하는 오류 & 실수 모음

### PUT 요청 관련

**`mutationFn`에 매개변수 두 개 전달 시도**
```ts
// ❌ React Query mutationFn은 매개변수 하나만 받음
const updateCar = async (url: string, car: Car) => { ... }

// ✅ 하나의 객체로 묶어서 전달 (CarEntry 타입 활용)
const updateCar = async (carEntry: CarEntry) => { ... }
```

**모달 열릴 때 기존 데이터가 안 채워지는 경우**
```tsx
// ❌ setOpen만 하고 setCar를 안 한 경우
const handleClickOpen = () => {
  setOpen(true); // input이 비어있음
}

// ✅ setCar로 기존 데이터 먼저 저장 후 open
const handleClickOpen = () => {
  setCar({ brand: cardata.brand, ... });
  setOpen(true);
}
```

**`Content-Type` 헤더 누락으로 PUT 요청 실패**
```ts
// ❌ 헤더 없이 보내면 Spring Boot에서 파싱 실패할 수 있음
await axios.put(url, car);

// ✅ Content-Type 명시
await axios.put(url, car, {
  headers: { 'Content-Type': 'application/json' }
});
```

**`?size=100` 없이 GET 요청 시 데이터가 잘려서 오는 경우**
```ts
// ❌ Spring Boot 기본 페이지 크기는 20개
axios.get(`${BASE_URL}/api/vehicles`);

// ✅ size 파라미터로 전체 조회
axios.get(`${BASE_URL}/api/vehicles?size=100`);
```

### 타입 관련

**`CarEntry` import 위치 혼동**
```ts
// ✅ CarEntry는 types.ts에 정의되어 있음
import { CarEntry } from "../types";
```

**`EditCar`의 props 타입 불일치**
```tsx
// ❌ CarResponse를 직접 타입으로 쓰면 구조분해 불가
export default function EditCar({ cardata }: CarResponse) { ... }

// ✅ FormProps로 감싸서 정의
type FormProps = { cardata: CarResponse }

```

### CarDialogContent 관련

**`name` 속성 누락**
```tsx
// ❌ name이 없으면 [event.target.name]이 undefined
<TextField label="Brand" value={car.brand} onChange={handleChange} />

// ✅ name은 Car 타입의 key와 반드시 일치해야 함
<TextField label="Brand" name="brand" value={car.brand} onChange={handleChange} />
```
