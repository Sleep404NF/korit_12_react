# GitHub API + React Query 학습 정리

## 1. GitHub REST API + axios + TypeScript

### 프로젝트 세팅
```bash
# React + TypeScript 프로젝트 생성 후
npm install axios
```

GitHub 리포지토리 검색 API 엔드포인트:
```
https://api.github.com/search/repositories?q={검색어}
```

---

### axios에서 제네릭 타입 적용

```tsx
import axios from 'axios';

type MyDataType = {
  id: number;
  name: string;
}

axios.get<MyDataType>(apiUrl)
  .then(response => console.log(response.data))
  .catch(error => console.log(error));
```

> **주의**: TypeScript는 JS로 컴파일되면 타입 정보가 제거된다.  
> 따라서 REST API가 예상과 다른 데이터를 반환할 경우 **런타임 오류로 포착되지 않는다**.  
> (빨간줄은 없는데 브라우저에서 반값이 나오는 경우가 이 때문)

---

### 전체 구현 코드

```tsx
import { useState } from 'react'
import './App.css'
import axios from 'axios';

type Repository = {
  id: number;       // .map() key 용도
  full_name: string;
  html_url: string;
}

function App() {
  const [keyword, setKeyword] = useState('');
  const [repodata, setRepodata] = useState<Repository[]>([]);

  const handleClick = () => {
    axios.get<{ items: Repository[] }>(
      `https://api.github.com/search/repositories?q=${keyword}`
    )
    .then(response => setRepodata(response.data.items))
    .catch(error => console.log(error));
  }

  return (
    <>
      <input type="text" onChange={(e) => setKeyword(e.target.value)} />
      <button onClick={handleClick}>검색</button>

      {repodata.length === 0 ? (
        <p>검색 결과가 없습니다.</p>
      ) : (
        <table>
          <tbody>
            {repodata.map(repo => (
              <tr key={repo.id}>
                <td>{repo.full_name}</td>
                <td>
                  <a href={repo.html_url}>{repo.full_name}의 레포지토리</a>
                </td>
              </tr>
            ))}
          </tbody>
        </table>
      )}
    </>
  )
}

export default App
```

### 핵심 포인트

- `useEffect()` 대신 버튼 클릭 시 API 호출 → 최초 렌더링 시점엔 키워드가 없기 때문
- `onClick` 핸들러에 체이닝 메서드가 필요하므로 `handleClick` 함수로 분리
- `axios.get<{ items: Repository[] }>` 처럼 응답 데이터 전체 구조를 제네릭으로 명시

## 2. React Query 라이브러리

### 프로젝트 세팅
```bash
npm install axios
npm install @tanstack/react-query@4
```

---

### React Query란?

서버 ↔ 클라이언트 간 데이터 패칭, 동기화, 캐싱을 쉽게 처리해주는 React 라이브러리.

> **캐싱(Caching)**: 동일한 데이터를 반복해서 가져오는 것을 줄이기 위해 메모리에 저장해두는 것

#### 사용 이유
서버 데이터는 시간이 지나면 변할 수 있어 항상 신선(fresh)하다고 보장할 수 없다.  
이 최신 상태 유지 로직을 클라이언트에서 직접 구현하는 부담을 덜어준다.

---

### 기본 세팅 (main.jsx)

```jsx
import React from 'react'
import ReactDOM from 'react-dom/client'
import App from './App.jsx'
import { QueryClient, QueryClientProvider } from '@tanstack/react-query'
import './index.css'

const queryClient = new QueryClient();

ReactDOM.createRoot(document.getElementById('root')).render(
  <React.StrictMode>
    <QueryClientProvider client={queryClient}>
      <App />
    </QueryClientProvider>
  </React.StrictMode>,
)
```

> `QueryClientProvider`를 최상단에 감싸줘야 하위 컴포넌트에서 React Query를 사용할 수 있다.  
> App.jsx 내부에서 감싸는 것도 가능하다.

## 3. useQuery

### 형식

```jsx
import { useQuery } from '@tanstack/react-query'

const { data, isLoading, error } = useQuery(queryKey, queryFn, options);

// v4 객체 형식 (권장)
const { isLoading, isError, data } = useQuery({
  queryKey: ['posts'],
  queryFn: fetchPosts,
});
```

### argument 설명

| 항목 | 설명 |
|------|------|
| `queryKey` | 쿼리마다 부여하는 고유 키. 문자열 또는 배열로 지정. 캐싱/재요청에 활용됨 |
| `queryFn` | 데이터를 가져오는 함수. `Promise`를 return해야 함 |
| `options` | 생략 가능. `staleTime`, `retry`, `enabled` 등 |

### return 값 주요 속성

| 속성 | 설명 |
|------|------|
| `isLoading` | 데이터를 아직 받지 못한 상태 (`true`) |
| `isError` | 쿼리가 오류로 종료된 상태 |
| `isSuccess` | 쿼리 성공, 데이터 사용 가능 |
| `data` | 응답 데이터 |

---

### 사용 예시

```jsx
// api.js
export const fetchPosts = async () => {
  const res = await fetch('https://jsonplaceholder.typicode.com/posts');
  const data = await res.json();
  return data;
}

// App.jsx
import { useQuery } from '@tanstack/react-query'
import { fetchPosts } from './api'

function App() {
  const { isLoading, data: posts } = useQuery('posts', fetchPosts);

  return (
    <>
      {isLoading
        ? '로딩중...'
        : posts.map((post) => <div key={post.id}>{`${post.id} : ${post.title}`}</div>)
      }
    </>
  )
}
```

### GitHub API + useQuery 적용 예시

```jsx
// App.jsx
import './App.css'
import { QueryClient, QueryClientProvider } from '@tanstack/react-query'
import Repositories from './Repositories'

const queryClient = new QueryClient();

function App() {
  return (
    <QueryClientProvider client={queryClient}>
      <Repositories />
    </QueryClientProvider>
  )
}

export default App

// Repositories.jsx
import axios from 'axios'
import { useQuery } from '@tanstack/react-query'

export default function Repositories() {

  const getRepositories = async () => {
    const response = await axios.get(
      'https://api.github.com/search/repositories?q=korit_12'
    );
    return response.data.items;
  }

  const { isLoading, isError, data } = useQuery({
    queryKey: ['repositories'],
    queryFn: getRepositories,
  });

  if (isLoading) return <p>로딩중...</p>
  if (isError) return <p>오류가 발생했습니다.</p>

  return (
    <table>
      <tbody>
        {data.map(repo => (
          <tr key={repo.id}>
            <td>{repo.full_name}</td>
            <td>
              <a href={repo.html_url}>{repo.full_name}의 repository</a>
            </td>
          </tr>
        ))}
      </tbody>
    </table>
  );
}
```

### useQuery 사용의 강점 (axios 직접 사용과 비교)

| 항목 | axios 직접 사용 | React Query (useQuery) |
|------|----------------|------------------------|
| 상태 관리 | `useState` 필요 | **불필요** (data가 대신함) |
| 최초 1회 실행 | `useEffect(()=>{},[])` 필요 | **불필요** |
| 타입 제한 | 사용할 필드만 타입 정의 | **응답 전체 데이터 사용 가능** |
| 캐싱 | 직접 구현 필요 | **자동 지원** |

> 실제 프로젝트에서는 `useState` / `useEffect` / `useQuery` 모두 함께 사용하게 된다.

## 4. staleTime / cacheTime 옵션

```jsx
const { isLoading, isError, data } = useQuery({
  queryKey: ['repositories'],
  queryFn: getRepositories,
  staleTime: 60 * 1000, // 1분 (밀리초 단위)
});
```

| 옵션 | 설명 | 기본값 |
|------|------|--------|
| `staleTime` | 데이터가 최신으로 유지되는 시간. 이 시간 동안은 재요청 안 함 | `0` (즉시 stale) |
| `cacheTime` | 비활성 쿼리가 가비지 컬렉션 대상이 되는 시간 | `5분` |

> 재포커스 시 자동 재요청이 너무 잦다면 `staleTime`을 설정하여 불필요한 재요청을 방지할 수 있다.

## 5. useMutation

### 형식

```jsx
import { useMutation } from '@tanstack/react-query'

const { data, isLoading, mutate } = useMutation(mutationFn, options);
```

- `useQuery`가 **GET** 요청용이라면, `useMutation`은 **CUD(생성·수정·삭제)** 요청용이다.
- `mutate`는 `mutationFn`을 실행시키는 **트리거 함수**이다.

### options 주요 콜백

| 옵션 | 실행 시점 |
|------|----------|
| `onMutate` | mutation 함수 실행 **전** |
| `onSuccess` | mutation **성공** 시 |
| `onError` | mutation **실패** 시 |
| `onSettled` | 성공/실패 **무관하게** 항상 실행 |

### 사용 예시

```jsx
import { useMutation } from '@tanstack/react-query'
import axios from 'axios'

const mutation = useMutation({
  mutationFn: newCar => {
    return axios.post('api/cars', newCar);
  },
  onSuccess: (data, variables, context) => {
    // 변경 성공 시 UI 업데이트 또는 toast 메세지 등
  },
  onError: (error, variables, context) => {
    // 오류 발생 시 처리 로직
  },
});

// 호출 방법
mutation.mutate(newCar);
```

> `mutation.mutate()`로 실제 호출한다는 점을 잊지 말 것.  
> 정의만 해놓고 호출을 빠뜨리는 실수가 자주 발생한다.
