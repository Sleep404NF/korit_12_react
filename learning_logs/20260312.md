## 1. 기본 레이아웃 구성

```tsx
// App.tsx
import { Container, AppBar, Toolbar, Typography, CssBaseline } from "@mui/material"

function App() {
  return (
    <Container maxWidth='xl'>
      <CssBaseline />
      <AppBar position="static">
        <Toolbar>
          <Typography variant="h6">
            Car Shop
          </Typography>
        </Toolbar>
      </AppBar>
    </Container>
  )
}
```

### 주요 포인트

**AppBar의 `position` prop**

| 값 | 동작 |
|----|------|
| `static` | 스크롤 시 앱 바가 함께 사라짐 |
| `fixed` | 스크롤해도 상단에 고정 유지 |

**`CssBaseline`**
- 브라우저마다 다른 기본 CSS를 초기화하여 균일한 UI를 유지시켜주는 컴포넌트.
- 최상단에 위치시켜 하위 컴포넌트 전체에 영향을 주도록 한다.

**`maxWidth`**
- `Container`의 최대 너비 지정. `'xl'`, `'lg'` 등 문자열 축약어로 값을 받는다.

> **attribute vs props**: HTML에서는 `attribute`, React에서는 `props` (properties의 줄임말).  
> Props Drilling 개념과 연결되므로 구분해서 기억할 것.

## 2. 타입 정의

Spring Boot REST API는 HATEOAS 방식으로 응답하기 때문에 일반 JSON과 달리 `_links`, `_embedded` 같은 필드가 포함된다.

```ts
// types.ts
export type CarResponse = {
  brand: string;
  model: string;
  color: string;
  registrationNumber: string;
  modelYear: number;
  price: number;
  _links: {
    self: {
      href: string;
    };
    car: {
      href: string;
    };
    owner: {
      href: string;
    };
  };
}
```

> `_links.self.href`는 각 자동차 데이터의 고유 URL이다.  
> 이후 `.map()`의 `key` prop으로 활용한다.

## 3. GET 요청 — Carlist 컴포넌트

```tsx
// components/Carlist.tsx
import { useQuery } from "@tanstack/react-query";
import { CarResponse } from "../types";
import axios from "axios";

export default function Carlist() {

  const getCars = async (): Promise<CarResponse[]> => {
    const response = await axios.get('http://localhost:8080/api/vehicles');
    return response.data._embedded.cars;
  }

  const { data, error, isSuccess } = useQuery({
    queryKey: ['cars'],
    queryFn: getCars,
  });

  if (!isSuccess) {
    return <span>Loading...</span>
  } else if (error) {
    return <span>자동차 데이터를 가져오던 중 오류가 발생했습니다.</span>
  } else {
    return (
      <table>
        <tbody>
          {data.map((car: CarResponse) => (
            <tr key={car._links.self.href}>
              <td>{car.brand}</td>
              <td>{car.model}</td>
              <td>{car.color}</td>
              <td>{car.registrationNumber}</td>
              <td>{car.modelYear}</td>
              <td>{car.price}</td>
            </tr>
          ))}
        </tbody>
      </table>
    );
  }
}
```

### 핵심 포인트

**Spring Boot REST 응답 구조**
```json
{
  "_embedded": {
    "cars": [ {...}, {...} ]
  }
}
```
- 일반 API와 달리 데이터가 `_embedded.cars` 안에 중첩되어 있다.
- 그래서 `response.data` 가 아니라 `response.data._embedded.cars`로 접근한다.

**`getCars`의 반환 타입 명시**
```tsx
const getCars = async (): Promise<CarResponse[]> => { ... }
```
- `async` 함수는 항상 `Promise`를 반환하므로, 반환 타입을 `Promise<CarResponse[]>`로 명시한다.
- 이전에 배운 TS 제네릭 문법의 연장선.

**`key` prop에 `_links.self.href` 사용**
- 각 차량의 고유 URL이기 때문에 중복 없이 안전하게 `key`로 활용 가능.

## 4. QueryClientProvider 위치

```tsx
// App.tsx
import { QueryClient, QueryClientProvider } from "@tanstack/react-query"
import Carlist from "./components/Carlist";

const queryClient = new QueryClient();

function App() {
  return (
    <Container maxWidth='xl'>
      <CssBaseline />
      <AppBar position="static">
        <Toolbar>
          <Typography variant="h6">Car Shop</Typography>
        </Toolbar>
      </AppBar>
      <QueryClientProvider client={queryClient}>
        <Carlist />
      </QueryClientProvider>
    </Container>
  )
}
```

> `QueryClientProvider`는 `useQuery`를 사용하는 컴포넌트를 감싸야 한다.  
> `AppBar`는 데이터와 무관하므로 Provider 밖에 위치시킨다.

## 5. 자주 발생하는 오류 & 실수 모음

### Spring Boot 연동 관련

**CORS 오류**
```
Access to XMLHttpRequest at 'http://localhost:8080/api/vehicles' 
from origin 'http://localhost:5173' has been blocked by CORS policy
```
- 프론트(5173포트)와 백엔드(8080포트)가 달라서 발생.
- Spring Boot 쪽에서 `@CrossOrigin` 어노테이션이 빠졌거나 설정이 잘못된 경우.
- 백엔드 담당자에게 확인하거나 직접 수정해야 한다.

**`_embedded` 접근 오류**
```
TypeError: Cannot read properties of undefined (reading 'cars')
```
- `response.data.cars`로 잘못 접근했을 때 발생.
- Spring Boot HATEOAS 응답은 반드시 `response.data._embedded.cars`로 접근해야 한다.
- API 응답 구조를 브라우저 네트워크 탭 또는 Postman으로 먼저 확인하는 습관을 들일 것.

**백엔드 서버 미실행**
```
Network Error / ERR_CONNECTION_REFUSED
```
- Spring Boot 서버를 먼저 실행하지 않은 경우.
- `http://localhost:8080/api/vehicles`를 브라우저에서 직접 열어서 응답이 오는지 확인할 것.

---

### React Query 관련

**`QueryClientProvider` 범위 밖에서 `useQuery` 사용**
```
Error: No QueryClient set, use QueryClientProvider to set one
```
- `useQuery`를 사용하는 컴포넌트가 `QueryClientProvider` 밖에 있을 때 발생.
- `Carlist`가 `QueryClientProvider` 내부에 위치하는지 확인할 것.

**`queryClient` 인스턴스를 컴포넌트 안에서 생성**
```tsx
// ❌ 잘못된 예 — 렌더링마다 새 인스턴스 생성됨
function App() {
  const queryClient = new QueryClient(); // 컴포넌트 내부에 있으면 안됨
}

// ✅ 올바른 예 — 컴포넌트 외부에서 생성
const queryClient = new QueryClient();
function App() { ... }
```

---

### TypeScript 관련

**`_links` 타입 누락**
```tsx
// ❌ _links 없이 타입 정의 시
type CarResponse = {
  brand: string;
  model: string;
}

// .map()에서 key로 car._links.self.href 사용하면 TS 오류 발생
```
- `_links`까지 타입에 포함시켜야 `key` prop으로 사용할 수 있다.

**`getCars` 반환 타입 불일치**
```tsx
// ❌ _embedded 구조를 무시하고 반환하면 타입 오류
const getCars = async (): Promise<CarResponse[]> => {
  const response = await axios.get('...');
  return response.data; // _embedded.cars가 빠짐
}
```
- 반환값이 `CarResponse[]`여야 하는데 `_embedded` 객체 전체를 반환하면 타입 불일치.
- 반드시 `response.data._embedded.cars`를 반환해야 한다.

---

### 조건부 렌더링 관련

**`isSuccess` / `error` 체크 순서 주의**
```tsx
// ❌ error를 먼저 체크하면 isSuccess가 false인 로딩 중에도
//    error가 없으면 data에 접근하려다 오류 발생 가능
if (error) { ... }
if (!isSuccess) { ... }

// ✅ 로딩 상태를 먼저 처리하는 것이 안전
if (!isSuccess) { return <span>Loading...</span> }
if (error) { return <span>오류 발생</span> }
```
