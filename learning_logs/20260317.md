# Frontend Protection + OAuth2 설정 학습 정리

---

## 1. 복습 — 프론트엔드 CRUD (axios)

### axios 요청 메서드 정리

| 요청 | 메서드 | 필요한 것 |
|------|--------|----------|
| GET (전체) | `axios.get(url)` | URL만 |
| GET (개별) | `axios.get(url)` | URL + ID (고유값) |
| DELETE | `axios.delete(url)` | URL + ID |
| POST | `axios.post(url, data, config)` | URL + 객체 + 헤더 |
| PUT | `axios.put(url, data, config)` | URL + ID + 객체 + 헤더 |

### POST / PUT argument 상세

```ts
// POST
axios.post(
  'http://localhost:8080/api/vehicles',               // arg1: URL
  car,                                                // arg2: 추가할 데이터 객체
  { headers: { 'Content-Type': 'application/json' } } // arg3: 메타데이터
)

// PUT
axios.put(
  'http://localhost:8080/api/vehicles/1',             // arg1: URL + ID (수정 대상 고유값)
  car,                                                // arg2: 수정할 전체 데이터 객체
  { headers: { 'Content-Type': 'application/json' } } // arg3: 메타데이터
)
```

> PUT은 수정 대상의 **전체 필드**가 포함된 객체를 보내야 한다.  
> 일부 필드만 보내는 부분 수정은 PATCH 요청을 사용한다.

## 2. 현재 상황 — Spring Security 적용

백엔드 `SecurityConfig` 수정으로 `/login` POST 요청을 제외한  
**모든 엔드포인트에 인증이 필요**한 상태.

| 환경 | 인증 방법 |
|------|----------|
| 프론트엔드 서버 | 서버 실행 시 자동 처리 |
| Postman | `/login`으로 JWT 발급 후 Authorization 탭에 붙여넣기 |
| HeidiSQL | DB 최초 접속 시 username/password 인증 완료로 별도 처리 불필요 |

### Postman JWT 발급 순서

```
POST /login
  body: { username: "user", password: "user" }
         또는
         { username: "admin", password: "admin" }
  → JWT 발급
    → Authorization 탭에 복붙
      → 이후 요청 가능
```

## 3. Frontend Protection — JWT 인증 흐름

```
사용자 로그인
  → /login 엔드포인트로 POST 요청
    → 응답 헤더의 Authorization에서 JWT 추출
      → localStorage에 저장
        → 이후 모든 요청의 headers에 JWT 포함
          → Spring Security 인증 통과
```

## 4. Login 컴포넌트

### handleLogin — 동기 방식 (async/await)

```tsx
const handleLogin = async () => {
  try {
    const response = await axios.post(
      `${import.meta.env.VITE_API_URL}/login`,
      user,
      { headers: { 'Content-Type': 'application/json' } }
    );

    const jwtToken = response.headers.authorization;

    if (jwtToken !== null && jwtToken !== undefined) {
      localStorage.setItem('jwt', jwtToken); // 브라우저에 JWT 저장
      setAuth(true);                         // 인증 상태 true로 변경
    }
  } catch (err) {
    console.error('로그인 중 문제가 발생했습니다: ', err);
  }
};
```

### handleLogin — 비동기 방식 (.then())

```tsx
const handleLogin = () => {
  axios.post(import.meta.env.VITE_API_URL + '/login', user, {
    headers: { 'Content-Type': 'application/json' }
  })
  .then(res => {
    const jwtToken = res.headers.authorization;
    if (jwtToken !== null && jwtToken !== undefined) {
      localStorage.setItem('jwt', jwtToken);
      setAuth(true);
    }
  })
  .catch(() => setOpen(true)); // 로그인 실패 시 Snackbar 표시
};
```

> 두 방식은 동일한 동작을 한다. 취향에 따라 선택하면 된다.  
> `.catch(err => console.error(err))`를 `.catch(() => setOpen(true))`로 바꾸면  
> 오류 발생 시 콘솔 대신 토스트 메시지로 사용자에게 피드백을 줄 수 있다.

### Login.tsx 전체 코드

```tsx
import { useState } from "react";
import axios from "axios";
import { Button, TextField, Stack, Snackbar } from "@mui/material";
import Carlist from "./Carlist";

type User = {
  username: string;
  password: string;
}

export default function Login() {
  const [open, setOpen] = useState(false);
  const [user, setUser] = useState<User>({
    username: '',
    password: ''
  });
  const [isAuthenticated, setAuth] = useState(false);

  const handleChange = (event: React.ChangeEvent<HTMLInputElement>) => {
    setUser({ ...user, [event.target.name]: event.target.value });
  }

  const handleLogin = () => {
    axios.post(import.meta.env.VITE_API_URL + '/login', user, {
      headers: { 'Content-Type': 'application/json' }
    })
    .then(res => {
      const jwtToken = res.headers.authorization;
      if (jwtToken !== null && jwtToken !== undefined) {
        localStorage.setItem('jwt', jwtToken);
        setAuth(true);
      }
    })
    .catch(() => setOpen(true));
  };

  const handleLogout = () => {
    setAuth(false);
    localStorage.setItem('jwt', ''); // JWT 초기화
  }

  // 인증 완료 시 Carlist 렌더링, 미완료 시 로그인 폼 렌더링
  if (isAuthenticated) {
    return <Carlist logout={handleLogout} />;
  }

  return (
    <>
      <Snackbar
        open={open}
        autoHideDuration={3000}
        onClose={() => setOpen(false)}
        message='ID 혹은 비밀번호가 틀렸습니다.'
      />
      <Stack spacing={2} alignItems='center' mt={2}>
        <TextField name='username' label='Username' onChange={handleChange} />
        <TextField name='password' label='Password' onChange={handleChange} />
        <Button variant="outlined" color='primary' onClick={handleLogin}>
          Login
        </Button>
      </Stack>
    </>
  );
}
```

## 5. JWT를 포함한 axios 요청

### 개별 함수마다 token 추가 (리팩토링 전)

```ts
export const getCars = async (): Promise<CarResponse[]> => {
  const token = localStorage.getItem('jwt'); // 매 함수마다 반복
  const response = await axios.get(
    `${import.meta.env.VITE_API_URL}/api/vehicles`,
    { headers: { 'Authorization': token } }
  );
  return response.data._embedded.cars;
}
```

> 모든 함수마다 `localStorage.getItem('jwt')`가 반복되는 문제 발생.

---

### getAxiosConfig()로 리팩토링 (최종 코드)

```ts
import { CarEntry, CarResponse, Car } from '../types'
import axios, { AxiosRequestConfig } from 'axios'

// 공통 헤더 설정 함수 — 모든 요청에서 재사용
const getAxiosConfig = (): AxiosRequestConfig => {
  const token = localStorage.getItem('jwt');
  return {
    headers: {
      'Authorization': token,
      'Content-Type': 'application/json'
    }
  };
}

// GET
export const getCars = async (): Promise<CarResponse[]> => {
  const response = await axios.get(
    `${import.meta.env.VITE_API_URL}/api/vehicles?size=100`,
    getAxiosConfig()
  );
  return response.data._embedded.cars;
}

// DELETE
export const deleteCar = async (link: string) => {
  const response = await axios.delete(link, getAxiosConfig());
  return response.data;
}

// POST
export const addCar = async (car: Car) => {
  const response = await axios.post(
    `${import.meta.env.VITE_API_URL}/api/vehicles`,
    car,
    getAxiosConfig()
  );
  return response.data;
}

// PUT
export const updateCar = async (carEntry: CarEntry): Promise<CarResponse> => {
  const response = await axios.patch(carEntry.url, carEntry.car, getAxiosConfig());
  return response.data;
}
```

### AxiosRequestConfig란?

Axios 라이브러리에서 HTTP 요청을 만들 때 사용하는 구성 객체의 타입.  
`headers` 외에도 `timeout`, `params`, `baseURL` 등 다양한 옵션을 담을 수 있는 일종의 interface에 해당한다.

```ts
// 반환 타입을 명시하면 타입 안정성 확보
const getAxiosConfig = (): AxiosRequestConfig => { ... }
```

## 6. 로그아웃 (Logout)

```ts
const handleLogout = () => {
  setAuth(false);
  localStorage.setItem('jwt', ''); // JWT 빈 문자열로 초기화
}
```

> `localStorage.removeItem('jwt')`로 완전히 삭제하는 것도 가능하다.  
> 빈 문자열로 설정하는 방식과 달리 key 자체가 사라진다.

### 로그아웃 버튼 위치와 Props Drilling

현재 구조에서는 `handleLogout`이 `Login`에 정의되어 있고 `Carlist`로 props drilling 된다.

```tsx
// Login.tsx — logout 함수를 Carlist로 전달
if (isAuthenticated) {
  return <Carlist logout={handleLogout} />;
}

// Carlist.tsx — logout prop 받기
type CarlistProps = {
  logout?: () => void;
}

export default function Carlist({ logout }: CarlistProps) {
  return (
    <Stack direction='row' alignItems='center' justifyContent='space-between'>
      <AddCar />
      <Button onClick={logout}>Log out</Button>
    </Stack>
    // ...
  )
}
```

### 페이지가 복잡해질 경우

AppBar에 로그아웃 버튼을 두고 여러 페이지에 표시하려면  
Props Drilling만으로는 한계가 있다. 이럴 때는 **전역 상태 관리**가 필요하다.

| 라이브러리 | 특징 |
|-----------|------|
| Context API | React 내장. 간단한 전역 상태에 적합 |
| Zustand | 외부 라이브러리. 간결하고 성능 좋음. 복잡한 상태 관리에 적합 |

> 예시: 게시물 좋아요 수를 마이페이지에서도 표시하려면  
> `like` 상태도 전역으로 관리해야 한다.

## 7. Carlist.tsx 최종 코드

```tsx
import { useQuery, useMutation, useQueryClient } from "@tanstack/react-query";
import { getCars, deleteCar } from "../api/carapi";
import { DataGrid, GridColDef, GridCellParams, GridToolbar } from "@mui/x-data-grid";
import { Snackbar, IconButton, Tooltip, Button, Stack } from "@mui/material";
import DeleteForeverRoundedIcon from '@mui/icons-material/DeleteForeverRounded';
import { useState } from "react";
import AddCar from "./AddCar";
import EditCar from "./EditCar";

type CarlistProps = {
  logout?: () => void;
}

export default function Carlist({ logout }: CarlistProps) {
  const [open, setOpen] = useState(false);
  const queryClient = useQueryClient();

  const { data, error, isSuccess } = useQuery({
    queryKey: ['cars'],
    queryFn: getCars
  });

  const { mutate } = useMutation(deleteCar, {
    onSuccess: () => {
      setOpen(true);
      queryClient.invalidateQueries({ queryKey: ['cars'] });
    },
    onError: err => console.log(err),
  });

  const columns: GridColDef[] = [
    { field: 'brand', headerName: 'Brand', width: 200 },
    { field: 'model', headerName: 'Model', width: 200 },
    { field: 'color', headerName: 'Color', width: 200 },
    { field: 'registrationNumber', headerName: 'Reg.No.', width: 200 },
    { field: 'modelYear', headerName: 'Year', width: 200 },
    { field: 'price', headerName: 'Price', width: 200 },
    {
      field: 'edit',
      headerName: '',
      sortable: false,
      filterable: false,
      disableColumnMenu: true,
      renderCell: (params: GridCellParams) => <EditCar cardata={params.row} />
    },
    {
      field: 'delete',
      headerName: '',
      sortable: false,
      filterable: false,
      disableColumnMenu: true,
      renderCell: (params: GridCellParams) => (
        <Tooltip title='delete car'>
          <IconButton aria-label='delete' size='small'
            onClick={() => {
              if (confirm(`${params.row.brand}의 ${params.row.color} ${params.row.model}을(를) 삭제하시겠습니까?`)) {
                mutate(params.row._links.self.href);
              }
            }}
          >
            <DeleteForeverRoundedIcon fontSize="small" />
          </IconButton>
        </Tooltip>
      )
    },
  ]

  if (!isSuccess) return <span>Loading ... ⏱️</span>
  if (error) return <span>자동차 데이터를 가져오던 중 오류가 발생했습니다... 😂</span>

  return (
    <>
      <Stack direction='row' alignItems='center' justifyContent='space-between'>
        <AddCar />
        <Button onClick={logout}>Log out</Button>
      </Stack>
      <DataGrid
        rows={data}
        columns={columns}
        disableRowSelectionOnClick={true}
        getRowId={row => row._links.self.href}
        slots={{ toolbar: GridToolbar }}
      />
      <Snackbar
        open={open}
        autoHideDuration={2000}
        onClose={() => setOpen(false)}
        message='해당 차량 정보가 삭제되었습니다 🚔'
      />
    </>
  );
}
```

## 9. 자주 발생하는 오류 & 실수 모음

### JWT 관련

**로그인 성공했는데 Carlist에서 데이터가 안 불러와지는 경우**
```ts
// ❌ getCars()에 토큰이 포함되지 않은 경우
const response = await axios.get(`${BASE_URL}/api/vehicles`);

// ✅ getAxiosConfig()로 토큰 포함
const response = await axios.get(`${BASE_URL}/api/vehicles`, getAxiosConfig());
```

**JWT가 응답 헤더에 없는 경우**
```ts
// authorization 헤더 키는 소문자로 오는 경우가 많음
const jwtToken = res.headers['authorization']; // 소문자로 접근하는 것이 안전
```

**로그아웃 후에도 데이터가 보이는 경우**
```ts
// ❌ JWT만 지우고 isAuthenticated 상태를 안 바꾼 경우
localStorage.setItem('jwt', '');

// ✅ 상태도 함께 변경해야 Login 컴포넌트로 돌아감
setAuth(false);
localStorage.setItem('jwt', '');
```

---

### Props Drilling 관련

**`logout` prop이 undefined인 경우**
```tsx
// ❌ CarlistProps에 logout이 필수(required)로 정의된 경우
type CarlistProps = {
  logout: () => void; // 필수값이라 undefined면 오류
}

// ✅ 선택적(optional)으로 정의
type CarlistProps = {
  logout?: () => void; // ?로 선택적으로
}
```

---

### 코드 반복 관련

**함수마다 `localStorage.getItem('jwt')` 반복**
```ts
// ❌ 매 함수마다 토큰을 개별로 가져오는 경우
export const getCars = async () => {
  const token = localStorage.getItem('jwt');
  // ...
}
export const deleteCar = async () => {
  const token = localStorage.getItem('jwt'); // 반복
  // ...
}

// ✅ getAxiosConfig()로 공통화
const getAxiosConfig = (): AxiosRequestConfig => {
  const token = localStorage.getItem('jwt');
  return { headers: { 'Authorization': token, 'Content-Type': 'application/json' } };
}
```

## 오늘 학습 요약

1. **JWT 인증 흐름** — 로그인 → JWT 발급 → localStorage 저장 → 모든 요청 헤더에 포함
2. **handleLogin** — async/await 방식과 .then() 방식 두 가지로 작성
3. **getAxiosConfig()** — `AxiosRequestConfig` 타입으로 공통 헤더 설정 함수 분리
4. **로그아웃** — `setAuth(false)` + `localStorage.setItem('jwt', '')` 함께 처리
5. **Props Drilling 한계** — 전역 상태가 필요할 경우 Context API 또는 Zustand 사용
6. **OAuth2 프로젝트 세팅** — Spring Initializr 설정 및 Dependencies 구성
